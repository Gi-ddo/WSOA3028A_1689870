What is Semantic Markup? Why is it necessary?

According to modern-day dictionaries, semantic refers to the correct use and interpretation of a word or sentence. Using that definition, we can then define semantic markup by saying it is the correct use of a markup language like HTML to convey the correct use of HTML tags to give meaning to the type of content presented within the tags.

Semantic Markup is necessary because it helps browsers correctly identify the type of information the developers present on their websites. It is also useful because it is both human and machine-readable. If an automated bot were to check information presented in a website that used semantic markup the bot will be able to tell what each piece of information represents, this also applies to humans, the use of semantic markup makes it easier for other developers to understand what is being presented.

To give more context, the use of the div and span tags have no semantic meaning to the website, this makes it difficult for bots and humans to understand what different types of content represent. Therefore, the correct use of takes like section, article, and nav makes it easier to distinguish what each type of information represents. For example, the section tag represents a standalone section of content on the website and the nav tag represents navigation of links in a section of a page. The div and span tags do not provide this type of information and thus makes them asemantic tag for web development.
