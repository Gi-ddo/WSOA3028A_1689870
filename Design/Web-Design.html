Navigation


The though process behind the navigation in the website is to make sure that the user and still access all of the menu items even if they have accessed another one. The solution to this process was to implement a navigation bar on the top of every page to give users the option to access other menu items if they say wish to, this limits the amount of times they have to go back to the home page and access the memu screem just to access a different portion of the website. A back button has also been implemented on every page that will allow users to revist the last page they had access to before their current page.(More on this will be updated as progress on the website unfolds)

Current iteration of site (with CSS)


Assignment 3 Submission


So for this current iteration of the website I made a few changes with the overall design of the site. firstly I decided on the colour scheme of the site I have decided to used a neon colour style to highlight different aspects of my Website I used a few samples from here and here. this includes things like: the hover effect when you about to click on a link, while browsing the menu, the background colour and the colour of the menu tabs. I decided to use neon colours because they are very bright and colourful and I wanted to give off a steampunk setting in my website, which I feel I failed miserable at.


Another change I made to the cite was the way the menu structure works, so initially I had the main menu driven by Java script using a drop down menu systm, it worked fine for what it was at the time but I felt like I took too much off the screen and it did not scale well with my CSS implentation so I decided to remove that menu systm and replaced it with a HTML and CSS driven menu structure which is consistent through each page with subtle differnces in content but in future work I hope to brinf back the Java script menu structure.
